# StudyQuest 🎓

> An AI-powered study planner that tells students **what to study, when to study it** — and makes the whole process feel like a game.

Built at **Google Developer Group 2026** as a live demo of what's possible when you wire Gemini directly into a mobile-first study experience.

---

## What is this?

StudyQuest replaces the anxiety of "I have an exam in two weeks and I don't know where to start" with a structured, adaptive, gamified study plan — generated by AI in seconds.

The student provides a subject, their source materials (PDFs, URLs, notes), and a deadline. The AI does the rest: it maps the topic landscape, assesses existing knowledge via a short quiz, and builds a personalised level-by-level roadmap. Progress is gated by quizzes; if you fail a level, the schedule adjusts. Spaced repetition keeps earlier topics fresh.

It is **not** a chatbot or AI tutor. The student is in control. The AI is the coach in the background.

## Setup

### Prerequisites

- Node.js 18+
- `npm`

### Clone & install

```bash
git clone https://github.com/viganogabriele/gdg2026.git
cd gdg2026
npm install
```

### Environment variables

Copy the example and fill in your keys:

```bash
cp .env.example .env.local
```

| Variable | Where to get it | Required |
|---|---|---|
| `EXPO_PUBLIC_GEMINI_API_KEY` | [Google Cloud Console](https://console.cloud.google.com/) → APIs → Generative Language API | ✅ Primary |
| `EXPO_PUBLIC_OPEN_ROUTER_API_KEY` | [openrouter.ai/keys](https://openrouter.ai/keys) | ⬜ Fallback |

> If no keys are provided, the app will still work with mock data — but the AI-generated roadmap and quiz will be generic and not personalised to the subject.

### Run

```bash
npx expo start          # scan QR with Expo Go
```

## Architecture

```
app/
  (onboarding)/          # 6-step modal flow
    subject.tsx          #   1. What are you studying?
    sources.tsx          #   2. Add PDFs / URLs / notes
    deadline.tsx         #   3. Deadline + hours/week
    processing.tsx       #   4. AI analysis (animated)
    assessment.tsx       #   5. Knowledge quiz (12 Qs)
    roadmap-reveal.tsx   #   6. Animated roadmap preview
  (tabs)/
    index.tsx            # Home: today's objectives + streak
    roadmap.tsx          # Full level map with progress
    profile.tsx          # Stats, badges, settings
  quiz/[id].tsx          # Full-screen level quiz
  focus.tsx              # Landscape focus mode (topic + timer)
  spaced-review.tsx      # Daily spaced repetition session

services/
  gemini.ts              # AI provider layer
  api.ts                 # 3-tier fallback orchestration
  mockData.ts            # Deterministic mock generators
  spacedRepetition.ts    # Leitner box engine (5 boxes, local)
  notifications.ts       # Expo push notification scheduling

hooks/
  useStudyStore.ts       # Zustand store + AsyncStorage persistence
  useSpacedRepetition.ts # SR session management
  useGamePoints.ts       # Points / streak / badge triggers

constants/
  theme.ts               # Design tokens (colors, spacing, type)
  gamification.ts        # Point values, badge defs, level thresholds

types/index.ts           # All shared TypeScript types
```

### AI fallback chain

Every screen goes through `api.ts`, which tries three tiers in order:

```
api.generateAssessment()
  │
  ├─ 1. Gemini SDK  (@google/generative-ai)
  │      responseSchema + responseMimeType: application/json
  │      → type-safe structured output, no regex parsing
  │
  ├─ 2. OpenRouter  (REST, OpenAI-compatible)
  │      response_format: { type: "json_object" }
  │      → same model, different auth path
  │
  └─ 3. Mock data   (local, always available)
         → deterministic generators keyed by subject title
```
